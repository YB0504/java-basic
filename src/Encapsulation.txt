---------- 캡슐화 ----------
객체 지향 프로그래밍의 중요한 개념 중 하나
데이터와 데이터를 처리하는 메소드를 하나로
묶어서 외부에서 접근을 제한하는 것
캡슐화를 안전하게 완성하는 장치가 접근제어자!!!r

1. 데이터를 숨겨라
객체에는 속성과 기능이 있다. 캡슐화에서 가장 필수로 숨겨야하는 것은 속성(데이터)이다.
객체 외부에서 데이터를 직접 접근 가능하도록 하면 클래스 안에서
데이터를 다루는 로직을 무시하고 데이터를 변경할 수 있다. 캡슐화가 깨진다.
***** 객체의 데이터는 객체가 제공하는 기능인 메소드를 통해서 접근해야 한다. *****

2. 기능을 숨겨라
객체의 기능 중에서 외부에서 사용하지 않고 내부에서만 사용하는 기능을 숨기는 것이다.
***** 사용자 입장에서 꼭 필요한 기능만 외부에 노출 *****

접근제어자와 캡슐화를통해 데이터를 보호하는 것은 물론,
데이터와 메소드를 사용하는 개발자 입장에서 복잡도를 낮출 수 있다.
